## What the component does

The Leap Change component communicates state transitions to the user. It uses
both visual and semantic cues to ensure changes are perceivable by all users,
including those using assistive technologies.

## Design recommendations

- Ensure the change indicator has sufficient colour contrast against its
  background (minimum 3:1 ratio for graphical elements)
- Do not rely solely on colour to communicate the type of change; include a text
  label or icon as well
- Keep change descriptions concise and meaningful

## Development considerations

- Use appropriate ARIA live regions (`aria-live="polite"`) to announce changes
  to screen readers when content updates dynamically
- Ensure the component is keyboard accessible
- Provide meaningful labels that describe the nature of the change
