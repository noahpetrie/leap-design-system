## Overview

The Leap Change component provides a structured way to display modifications to
leave requests. Each change card shows the employee, leave type, what changed,
and when — with a colour-coded badge indicating the change type.

### When to use

- To display a history of changes on a leave request detail page
- In a manager dashboard showing recent leave request activity
- To show an audit trail of modifications to leave balances or dates

### When not to use

- For static leave information that has not changed
- As a replacement for toast notifications or alerts
- When displaying a simple list of leave requests without change context

## Change types

| Type | Usage |
| ---- | ----- |
| **Approved** | A leave request has been approved by a manager |
| **Denied** | A leave request has been denied, typically with a reason |
| **Modified** | Dates, duration, or leave type has been changed |
| **Pending** | A new request that is awaiting review |

## Formatting

### Anatomy

The Leap Change component consists of:

1. **Header** — employee name, leave type, and change type badge
2. **Before/after values** — shows the previous and new state with an arrow
3. **Timestamp** — when the change occurred
4. **Reason** (optional) — context for why the change was made

### Placement

Leap Change cards are typically stacked vertically in a change log or activity
feed. They work well inside side panels, detail pages, or dashboard widgets.
