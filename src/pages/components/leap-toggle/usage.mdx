## Overview

The Leap Toggle wraps Carbon's `Toggle` component and overrides the checked
state colour to use the Leap brand green (#0C8C5E) instead of Carbon's default
blue. All other behaviour, accessibility, and interaction patterns remain
identical to the Carbon Toggle.

### When to use

- For binary on/off settings in Leap applications
- To enable or disable features, preferences, or configurations
- When an immediate effect is expected (no submit button needed)

### When not to use

- When the user needs to select from more than two options — use a radio group
- When the action requires confirmation — use a checkbox with a submit button
- For triggering actions — use a button instead

## Formatting

### Anatomy

The Leap Toggle consists of:

1. **Label** — describes what the toggle controls
2. **Switch track** — the pill-shaped track that changes colour
3. **Switch handle** — the circular indicator that slides between states
4. **State label** (optional) — text showing the current state (On/Off)

### Colour behaviour

- **Off state**: Uses the default Carbon neutral colour
- **On state**: Uses Leap brand green (#0C8C5E)
- **Hover (on)**: Darkens to active green (#096B48)
- **Focus**: Green outline matching the brand
